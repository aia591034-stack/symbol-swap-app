========================================================================
Symbol アグリゲートトランザクション・物々交換テスト手順書
========================================================================

このプロジェクトは、ユーザーAとユーザーBが、運営（オペレーター）の手数料肩代わりのもと、
暗号化されたメッセージ（解凍鍵）を安全に同時交換するためのテスト環境です。

------------------------------------------------------------------------
1. 準備するもの
------------------------------------------------------------------------

■ 実行環境
- Node.js (v18以上推奨)
- npm (Node.jsに付属)

■ テストネット用アカウント（合計3つ）
テストを行うために、以下の3つのアカウントの「秘密鍵」が必要です。
1. 運営用（Operator）：手数料を支払うためのXYMが必要です。
2. ユーザーA：電子書籍の鍵を送る側。
3. ユーザーB：イラストの鍵を送る側。

※ アカウントを持っていない場合：
Symbol Wallet（デスクトップ版等）で「Testnet」アカウントを作成するか、
以下のツール等で使い捨ての鍵を生成してください。

■ テスト用XYMの入手
運営用アカウント（Operator）には、手数料として少額のXYMが必要です。
以下の「蛇口（Faucet）」サイトに、運営用のアドレスを入力してXYMを受け取ってください。
- Testnet Faucet: https://testnet.symbol.fyi/faucet

------------------------------------------------------------------------
2. セットアップ手順
------------------------------------------------------------------------

ターミナル（コマンドプロンプトやPowerShell）で以下のコマンドを実行します。

# 1. フォルダの作成
mkdir symbol-swap-app
cd symbol-swap-app

# 2. 初期化とライブラリのインストール
npm init -y
npm install symbol-sdk

# 3. スクリプトの作成
「swap.js」という名前でファイルを作成し、提供されたプログラムコードを貼り付けます。

# 4. 秘密鍵の書き換え
swap.js 内の以下の部分を、準備した実際の秘密鍵（64文字）に書き換えてください。
- operatorKeyPair
- userAKeyPair
- userBKeyPair

------------------------------------------------------------------------
3. テストの実行方法
------------------------------------------------------------------------

準備が整ったら、以下のコマンドでスクリプトを実行します。

node swap.js

実行すると、コンソールに以下のような情報が表示されます：
- 各ユーザーのアドレス
- 「トランザクション構築完了」のメッセージ
- 署名済みペイロード（バイナリデータの一部）

------------------------------------------------------------------------
4. 実行結果の確認方法（検証）
------------------------------------------------------------------------

実際にネットワークに送信（アナウンス）が成功した場合、以下の方法で確認できます。

1. Symbol ブロックエクスプローラーで確認
   https://testnet.symbol.fyi/
   検索窓に、実行後に表示された「Transaction Hash」を入力します。

2. 確認ポイント
   - Status: "Success" または "Confirmed" になっているか。
   - Signer: 運営（Operator）のアドレスになっているか（＝手数料を払った人）。
   - Inner Transactions: 2つのTransfer（転送）が含まれているか。
     - A -> B へのメッセージ
     - B -> A へのメッセージ

3. メッセージの解読テスト（発展）
   受信側の秘密鍵を使ってメッセージを復号し、
   「KEY_FOR_EBOOK_12345」といった文字列が正しく表示されるか確認します。
   ※Walletアプリにインポートすれば、UI上で復号して読むことができます。

------------------------------------------------------------------------
5. トラブルシューティング
------------------------------------------------------------------------

- 「Insufficient Balance」エラーが出る：
  運営アカウントにXYMが不足しています。Faucetで補充してください。
- 「Deadline passed」エラーが出る：
  PCの時刻がずれている可能性があります。設定で時刻を同期してください。
- 「Invalid Signature」エラーが出る：
  秘密鍵と公開鍵のペアが正しくないか、ネットワークタイプ（Testnet/Mainnet）の
  指定が間違っている可能性があります。
